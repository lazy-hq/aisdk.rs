title: AlibabaCn
---

AISDK provides first-class support for AlibabaCn with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the AlibabaCn provider feature:

```bash
cargo add aisdk --features alibaba-cn
```

This installs AISDK with the AlibabaCn provider enabled. Once you have enabled the AlibabaCn provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `AlibabaCn::model_name()`, where **model_name** is the AlibabaCn model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::AlibabaCn;

let alibaba-cn = AlibabaCn::deepseek_r1();
```

This initializes the provider with:

* Model: `"deepseek-r1"`
* API key from environment (if set with `DASHSCOPE_API_KEY`)
* AlibabaCn's default base URL (https://dashscope.aliyuncs.com/compatible-mode/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::AlibabaCn,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let alibaba-cn = AlibabaCn::deepseek_r1();

    let response = LanguageModelRequest::builder()
        .model(alibaba-cn)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `AlibabaCn::builder()`

### API Key

```rust
let alibaba-cn = AlibabaCn::<DeepseekR1>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `DASHSCOPE_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let alibaba-cn = AlibabaCn::<DeepseekR1>::builder()
    .base_url("https://dashscope.aliyuncs.com/compatible-mode/v1")
    .build()?;
```

### Path (Full URL Override)

Use `.path(...)` to override the full request URL instead of only the base URL.

```rust
let alibaba-cn = AlibabaCn::<DeepseekR1>::builder()
    .path("https://full-url.example/v1/chat/completions")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let alibaba-cn = AlibabaCn::<DeepseekR1>::builder()
    .provider_name("AlibabaCn")
    .build()?;
```

### Full Custom Configuration Example

```rust
let alibaba-cn = AlibabaCn::<DeepseekR1>::builder()
    .api_key("your-api-key")
    .base_url("https://dashscope.aliyuncs.com/compatible-mode/v1")
    .path("https://full-url.example/v1/chat/completions")
    .provider_name("AlibabaCn")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::AlibabaCn;

// Specify model as a string at runtime
let alibaba-cn = AlibabaCn::model_name("deepseek-r1");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::AlibabaCn,
};

let alibaba-cn = AlibabaCn::<DynamicModel>::builder()
	.model_name("deepseek-r1")
	.api_key("your-api-key")
	.base_url("https://dashscope.aliyuncs.com/compatible-mode/v1")
	.path("https://full-url.example/v1/chat/completions")
	.provider_name("AlibabaCn")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `AlibabaCn::deepseek_r1()`.
</Callout>
