title: GithubCopilot
---

AISDK provides first-class support for GithubCopilot with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the GithubCopilot provider feature:

```bash
cargo add aisdk --features github-copilot
```

This installs AISDK with the GithubCopilot provider enabled. Once you have enabled the GithubCopilot provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `GithubCopilot::model_name()`, where **model_name** is the GithubCopilot model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::GithubCopilot;

let github-copilot = GithubCopilot::claude_haiku_4_5();
```

This initializes the provider with:

* Model: `"claude-haiku-4.5"`
* API key from environment (if set with `GITHUB_TOKEN`)
* GithubCopilot's default base URL (https://api.githubcopilot.com)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::GithubCopilot,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let github-copilot = GithubCopilot::claude_haiku_4_5();

    let response = LanguageModelRequest::builder()
        .model(github-copilot)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `GithubCopilot::builder()`

### API Key

```rust
let github-copilot = GithubCopilot::<ClaudeHaiku45>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `GITHUB_TOKEN` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let github-copilot = GithubCopilot::<ClaudeHaiku45>::builder()
    .base_url("https://api.githubcopilot.com")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let github-copilot = GithubCopilot::<ClaudeHaiku45>::builder()
    .provider_name("GithubCopilot")
    .build()?;
```

### Full Custom Configuration Example

```rust
let github-copilot = GithubCopilot::<ClaudeHaiku45>::builder()
    .api_key("your-api-key")
    .base_url("https://api.githubcopilot.com")
    .provider_name("GithubCopilot")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::GithubCopilot;

// Specify model as a string at runtime
let github-copilot = GithubCopilot::model_name("claude-haiku-4.5");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::GithubCopilot,
};

let github-copilot = GithubCopilot::<DynamicModel>::builder()
	.model_name("claude-haiku-4.5")
	.api_key("your-api-key")
	.base_url("https://api.githubcopilot.com")
	.provider_name("GithubCopilot")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `GithubCopilot::claude_haiku_4_5()`.
</Callout>
