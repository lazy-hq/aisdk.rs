title: CloudflareWorkersAi
---

AISDK provides first-class support for CloudflareWorkersAi with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the CloudflareWorkersAi provider feature:

```bash
cargo add aisdk --features cloudflare-workers-ai
```

This installs AISDK with the CloudflareWorkersAi provider enabled. Once you have enabled the CloudflareWorkersAi provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `CloudflareWorkersAi::model_name()`, where **model_name** is the CloudflareWorkersAi model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::CloudflareWorkersAi;

let cloudflare-workers-ai = CloudflareWorkersAi::ai4bharat_indictrans2_en_indic_1b();
```

This initializes the provider with:

* Model: `"@cf/ai4bharat/indictrans2-en-indic-1B"`
* API key from environment (if set with `CLOUDFLARE_ACCOUNT_ID`)
* CloudflareWorkersAi's default base URL (https://api.cloudflare.com/client/v4/accounts/${CLOUDFLARE_ACCOUNT_ID}/ai/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::CloudflareWorkersAi,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let cloudflare-workers-ai = CloudflareWorkersAi::ai4bharat_indictrans2_en_indic_1b();

    let response = LanguageModelRequest::builder()
        .model(cloudflare-workers-ai)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `CloudflareWorkersAi::builder()`

### API Key

```rust
let cloudflare-workers-ai = CloudflareWorkersAi::<Ai4bharatIndictrans2EnIndic1b>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `CLOUDFLARE_ACCOUNT_ID` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let cloudflare-workers-ai = CloudflareWorkersAi::<Ai4bharatIndictrans2EnIndic1b>::builder()
    .base_url("https://api.cloudflare.com/client/v4/accounts/${CLOUDFLARE_ACCOUNT_ID}/ai/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let cloudflare-workers-ai = CloudflareWorkersAi::<Ai4bharatIndictrans2EnIndic1b>::builder()
    .provider_name("CloudflareWorkersAi")
    .build()?;
```

### Full Custom Configuration Example

```rust
let cloudflare-workers-ai = CloudflareWorkersAi::<Ai4bharatIndictrans2EnIndic1b>::builder()
    .api_key("your-api-key")
    .base_url("https://api.cloudflare.com/client/v4/accounts/${CLOUDFLARE_ACCOUNT_ID}/ai/v1")
    .provider_name("CloudflareWorkersAi")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::CloudflareWorkersAi;

// Specify model as a string at runtime
let cloudflare-workers-ai = CloudflareWorkersAi::model_name("@cf/ai4bharat/indictrans2-en-indic-1B");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::CloudflareWorkersAi,
};

let cloudflare-workers-ai = CloudflareWorkersAi::<DynamicModel>::builder()
    .model_name("@cf/ai4bharat/indictrans2-en-indic-1B")
    .api_key("your-api-key")
		.base_url("https://api.cloudflare.com/client/v4/accounts/${CLOUDFLARE_ACCOUNT_ID}/ai/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `CloudflareWorkersAi::ai4bharat_indictrans2_en_indic_1b()`.
</Callout>
