title: Inference

AISDK provides first-class support for Inference with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Inference provider feature:

```bash
cargo add aisdk --features inference
```

This installs AISDK with the Inference provider enabled. Once you have enabled the Inference provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Inference::model_name()`, where **model_name** is the Inference model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Inference;

let inference = Inference::google_gemma_3();
```

This initializes the provider with:

* Model: `"google/gemma-3"`
* API key from environment (if set with `INFERENCE_API_KEY`)
* Inference's default base URL (https://inference.net/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Inference,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let inference = Inference::google_gemma_3();

    let response = LanguageModelRequest::builder()
        .model(inference)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Inference::builder()`

### API Key

```rust
let inference = Inference::<GoogleGemma3>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `INFERENCE_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let inference = Inference::<GoogleGemma3>::builder()
    .base_url("https://inference.net/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let inference = Inference::<GoogleGemma3>::builder()
    .provider_name("Inference")
    .build()?;
```

### Full Custom Configuration Example

```rust
let inference = Inference::<GoogleGemma3>::builder()
    .api_key("your-api-key")
    .base_url("https://inference.net/v1")
    .provider_name("Inference")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Inference;

// Specify model as a string at runtime
let inference = Inference::model_name("google/gemma-3");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Inference,
};

let inference = Inference::<DynamicModel>::builder()
	.model_name("google/gemma-3")
	.api_key("your-api-key")
	.base_url("https://inference.net/v1")
	.provider_name("Inference")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Inference::google_gemma_3()`.
</Callout>
