title: Alibaba
---

AISDK provides first-class support for Alibaba with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Alibaba provider feature:

```bash
cargo add aisdk --features alibaba
```

This installs AISDK with the Alibaba provider enabled. Once you have enabled the Alibaba provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Alibaba::model_name()`, where **model_name** is the Alibaba model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Alibaba;

let alibaba = Alibaba::qvq_max();
```

This initializes the provider with:

* Model: `"qvq-max"`
* API key from environment (if set with `DASHSCOPE_API_KEY`)
* Alibaba's default base URL (https://dashscope-intl.aliyuncs.com/compatible-mode/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Alibaba,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let alibaba = Alibaba::qvq_max();

    let response = LanguageModelRequest::builder()
        .model(alibaba)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Alibaba::builder()`

### API Key

```rust
let alibaba = Alibaba::<QvqMax>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `DASHSCOPE_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let alibaba = Alibaba::<QvqMax>::builder()
    .base_url("https://dashscope-intl.aliyuncs.com/compatible-mode/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let alibaba = Alibaba::<QvqMax>::builder()
    .provider_name("Alibaba")
    .build()?;
```

### Full Custom Configuration Example

```rust
let alibaba = Alibaba::<QvqMax>::builder()
    .api_key("your-api-key")
    .base_url("https://dashscope-intl.aliyuncs.com/compatible-mode/v1")
    .provider_name("Alibaba")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Alibaba;

// Specify model as a string at runtime
let alibaba = Alibaba::model_name("qvq-max");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Alibaba,
};

let alibaba = Alibaba::<DynamicModel>::builder()
    .model_name("qvq-max")
    .api_key("your-api-key")
		.base_url("https://dashscope-intl.aliyuncs.com/compatible-mode/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Alibaba::qvq_max()`.
</Callout>
