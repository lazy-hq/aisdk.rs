title: ZaiCodingPlan
---

AISDK provides first-class support for ZaiCodingPlan with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the ZaiCodingPlan provider feature:

```bash
cargo add aisdk --features zai-coding-plan
```

This installs AISDK with the ZaiCodingPlan provider enabled. Once you have enabled the ZaiCodingPlan provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `ZaiCodingPlan::model_name()`, where **model_name** is the ZaiCodingPlan model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::ZaiCodingPlan;

let zai-coding-plan = ZaiCodingPlan::glm_4_5();
```

This initializes the provider with:

* Model: `"glm-4.5"`
* API key from environment (if set with `ZHIPU_API_KEY`)
* ZaiCodingPlan's default base URL (https://api.z.ai/api/coding/paas/v4)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::ZaiCodingPlan,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let zai-coding-plan = ZaiCodingPlan::glm_4_5();

    let response = LanguageModelRequest::builder()
        .model(zai-coding-plan)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `ZaiCodingPlan::builder()`

### API Key

```rust
let zai-coding-plan = ZaiCodingPlan::<Glm45>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `ZHIPU_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let zai-coding-plan = ZaiCodingPlan::<Glm45>::builder()
    .base_url("https://api.z.ai/api/coding/paas/v4")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let zai-coding-plan = ZaiCodingPlan::<Glm45>::builder()
    .provider_name("ZaiCodingPlan")
    .build()?;
```

### Full Custom Configuration Example

```rust
let zai-coding-plan = ZaiCodingPlan::<Glm45>::builder()
    .api_key("your-api-key")
    .base_url("https://api.z.ai/api/coding/paas/v4")
    .provider_name("ZaiCodingPlan")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::ZaiCodingPlan;

// Specify model as a string at runtime
let zai-coding-plan = ZaiCodingPlan::model_name("glm-4.5");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::ZaiCodingPlan,
};

let zai-coding-plan = ZaiCodingPlan::<DynamicModel>::builder()
	.model_name("glm-4.5")
	.api_key("your-api-key")
	.base_url("https://api.z.ai/api/coding/paas/v4")
	.provider_name("ZaiCodingPlan")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `ZaiCodingPlan::glm_4_5()`.
</Callout>
