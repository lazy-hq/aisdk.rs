title: DeepSeek
---

AISDK provides first-class support for DeepSeek with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the DeepSeek provider feature:

```bash
cargo add aisdk --features deepseek
```

This installs AISDK with the DeepSeek provider enabled. Once you have enabled the DeepSeek provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `DeepSeek::model_name()`, where **model_name** is the DeepSeek model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::DeepSeek;

let deepseek = DeepSeek::deepseek_chat();
```

This initializes the provider with:

* Model: `"deepseek-chat"`
* API key from environment (if set with `DEEPSEEK_API_KEY`)
* DeepSeek's default base URL (https://api.deepseek.com/)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::DeepSeek,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let deepseek = DeepSeek::deepseek_chat();

    let response = LanguageModelRequest::builder()
        .model(deepseek)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `DeepSeek::builder()`

### API Key

```rust
let deepseek = DeepSeek::<DeepseekChat>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `DEEPSEEK_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let deepseek = DeepSeek::<DeepseekChat>::builder()
    .base_url("https://api.deepseek.com/")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let deepseek = DeepSeek::<DeepseekChat>::builder()
    .provider_name("DeepSeek")
    .build()?;
```

### Full Custom Configuration Example

```rust
let deepseek = DeepSeek::<DeepseekChat>::builder()
    .api_key("your-api-key")
    .base_url("https://api.deepseek.com/")
    .provider_name("DeepSeek")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::DeepSeek;

// Specify model as a string at runtime
let deepseek = DeepSeek::model_name("deepseek-chat");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::DeepSeek,
};

let deepseek = DeepSeek::<DynamicModel>::builder()
    .model_name("deepseek-chat")
    .api_key("your-api-key")
		.base_url("https://api.deepseek.com/")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `DeepSeek::deepseek_chat()`.
</Callout>
