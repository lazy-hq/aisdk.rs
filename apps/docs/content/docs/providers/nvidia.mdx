title: Nvidia
---

AISDK provides first-class support for Nvidia with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Nvidia provider feature:

```bash
cargo add aisdk --features nvidia
```

This installs AISDK with the Nvidia provider enabled. Once you have enabled the Nvidia provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Nvidia::model_name()`, where **model_name** is the Nvidia model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Nvidia;

let nvidia = Nvidia::black_forest_labs_flux_1_dev();
```

This initializes the provider with:

* Model: `"black-forest-labs/flux.1-dev"`
* API key from environment (if set with `NVIDIA_API_KEY`)
* Nvidia's default base URL (https://integrate.api.nvidia.com/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Nvidia,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let nvidia = Nvidia::black_forest_labs_flux_1_dev();

    let response = LanguageModelRequest::builder()
        .model(nvidia)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Nvidia::builder()`

### API Key

```rust
let nvidia = Nvidia::<BlackForestLabsFlux1Dev>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `NVIDIA_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let nvidia = Nvidia::<BlackForestLabsFlux1Dev>::builder()
    .base_url("https://integrate.api.nvidia.com/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let nvidia = Nvidia::<BlackForestLabsFlux1Dev>::builder()
    .provider_name("Nvidia")
    .build()?;
```

### Full Custom Configuration Example

```rust
let nvidia = Nvidia::<BlackForestLabsFlux1Dev>::builder()
    .api_key("your-api-key")
    .base_url("https://integrate.api.nvidia.com/v1")
    .provider_name("Nvidia")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Nvidia;

// Specify model as a string at runtime
let nvidia = Nvidia::model_name("black-forest-labs/flux.1-dev");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Nvidia,
};

let nvidia = Nvidia::<DynamicModel>::builder()
    .model_name("black-forest-labs/flux.1-dev")
    .api_key("your-api-key")
		.base_url("https://integrate.api.nvidia.com/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Nvidia::black_forest_labs_flux_1_dev()`.
</Callout>
