AISDK provides first-class support for Chutes with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Chutes provider feature:

```bash
cargo add aisdk --features chutes
```

This installs AISDK with the Chutes provider enabled. Once you have enabled the Chutes provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Chutes::model_name()`, where **model_name** is the Chutes model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Chutes;

let chutes = Chutes::minimaxai_minimax_m2_1_tee();
```

This initializes the provider with:

* Model: `"MiniMaxAI/MiniMax-M2.1-TEE"`
* API key from environment (if set with `CHUTES_API_KEY`)
* Chutes's default base URL (https://llm.chutes.ai/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Chutes,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let chutes = Chutes::minimaxai_minimax_m2_1_tee();

    let response = LanguageModelRequest::builder()
        .model(chutes)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Chutes::builder()`

### API Key

```rust
let chutes = Chutes::<MinimaxaiMinimaxM21Tee>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `CHUTES_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let chutes = Chutes::<MinimaxaiMinimaxM21Tee>::builder()
    .base_url("https://llm.chutes.ai/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let chutes = Chutes::<MinimaxaiMinimaxM21Tee>::builder()
    .provider_name("Chutes")
    .build()?;
```

### Full Custom Configuration Example

```rust
let chutes = Chutes::<MinimaxaiMinimaxM21Tee>::builder()
    .api_key("your-api-key")
    .base_url("https://llm.chutes.ai/v1")
    .provider_name("Chutes")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Chutes;

// Specify model as a string at runtime
let chutes = Chutes::model_name("MiniMaxAI/MiniMax-M2.1-TEE");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Chutes,
};

let chutes = Chutes::<DynamicModel>::builder()
	.model_name("MiniMaxAI/MiniMax-M2.1-TEE")
	.api_key("your-api-key")
	.base_url("https://llm.chutes.ai/v1")
	.provider_name("Chutes")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Chutes::minimaxai_minimax_m2_1_tee()`.
</Callout>
