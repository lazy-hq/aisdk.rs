AISDK provides first-class support for Friendli with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Friendli provider feature:

```bash
cargo add aisdk --features friendli
```

This installs AISDK with the Friendli provider enabled. Once you have enabled the Friendli provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Friendli::model_name()`, where **model_name** is the Friendli model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Friendli;

let friendli = Friendli::lgai_exaone_exaone_4_0_1_32b();
```

This initializes the provider with:

* Model: `"LGAI-EXAONE/EXAONE-4.0.1-32B"`
* API key from environment (if set with `FRIENDLI_TOKEN`)
* Friendli's default base URL (https://api.friendli.ai/serverless/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Friendli,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let friendli = Friendli::lgai_exaone_exaone_4_0_1_32b();

    let response = LanguageModelRequest::builder()
        .model(friendli)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Friendli::builder()`

### API Key

```rust
let friendli = Friendli::<LgaiExaoneExaone40132b>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `FRIENDLI_TOKEN` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let friendli = Friendli::<LgaiExaoneExaone40132b>::builder()
    .base_url("https://api.friendli.ai/serverless/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let friendli = Friendli::<LgaiExaoneExaone40132b>::builder()
    .provider_name("Friendli")
    .build()?;
```

### Full Custom Configuration Example

```rust
let friendli = Friendli::<LgaiExaoneExaone40132b>::builder()
    .api_key("your-api-key")
    .base_url("https://api.friendli.ai/serverless/v1")
    .provider_name("Friendli")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Friendli;

// Specify model as a string at runtime
let friendli = Friendli::model_name("LGAI-EXAONE/EXAONE-4.0.1-32B");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Friendli,
};

let friendli = Friendli::<DynamicModel>::builder()
    .model_name("LGAI-EXAONE/EXAONE-4.0.1-32B")
    .api_key("your-api-key")
		.base_url("https://api.friendli.ai/serverless/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Friendli::lgai_exaone_exaone_4_0_1_32b()`.
</Callout>
