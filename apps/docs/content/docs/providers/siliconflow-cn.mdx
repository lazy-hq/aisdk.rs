title: SiliconflowCn
---

AISDK provides first-class support for SiliconflowCn with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the SiliconflowCn provider feature:

```bash
cargo add aisdk --features siliconflow-cn
```

This installs AISDK with the SiliconflowCn provider enabled. Once you have enabled the SiliconflowCn provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `SiliconflowCn::model_name()`, where **model_name** is the SiliconflowCn model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::SiliconflowCn;

let siliconflow-cn = SiliconflowCn::bytedance_seed_seed_oss_36b_instruct();
```

This initializes the provider with:

* Model: `"ByteDance-Seed/Seed-OSS-36B-Instruct"`
* API key from environment (if set with `SILICONFLOW_CN_API_KEY`)
* SiliconflowCn's default base URL (https://api.siliconflow.cn/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::SiliconflowCn,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let siliconflow-cn = SiliconflowCn::bytedance_seed_seed_oss_36b_instruct();

    let response = LanguageModelRequest::builder()
        .model(siliconflow-cn)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `SiliconflowCn::builder()`

### API Key

```rust
let siliconflow-cn = SiliconflowCn::<BytedanceSeedSeedOss36bInstruct>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `SILICONFLOW_CN_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let siliconflow-cn = SiliconflowCn::<BytedanceSeedSeedOss36bInstruct>::builder()
    .base_url("https://api.siliconflow.cn/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let siliconflow-cn = SiliconflowCn::<BytedanceSeedSeedOss36bInstruct>::builder()
    .provider_name("SiliconflowCn")
    .build()?;
```

### Full Custom Configuration Example

```rust
let siliconflow-cn = SiliconflowCn::<BytedanceSeedSeedOss36bInstruct>::builder()
    .api_key("your-api-key")
    .base_url("https://api.siliconflow.cn/v1")
    .provider_name("SiliconflowCn")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::SiliconflowCn;

// Specify model as a string at runtime
let siliconflow-cn = SiliconflowCn::model_name("ByteDance-Seed/Seed-OSS-36B-Instruct");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::SiliconflowCn,
};

let siliconflow-cn = SiliconflowCn::<DynamicModel>::builder()
    .model_name("ByteDance-Seed/Seed-OSS-36B-Instruct")
    .api_key("your-api-key")
		.base_url("https://api.siliconflow.cn/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `SiliconflowCn::bytedance_seed_seed_oss_36b_instruct()`.
</Callout>
