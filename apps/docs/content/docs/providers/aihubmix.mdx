title: Aihubmix

AISDK provides first-class support for Aihubmix with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Aihubmix provider feature:

```bash
cargo add aisdk --features aihubmix
```

This installs AISDK with the Aihubmix provider enabled. Once you have enabled the Aihubmix provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Aihubmix::model_name()`, where **model_name** is the Aihubmix model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Aihubmix;

let aihubmix = Aihubmix::kimi_k2_0905();
```

This initializes the provider with:

* Model: `"Kimi-K2-0905"`
* API key from environment (if set with `AIHUBMIX_API_KEY`)
* Aihubmix's default base URL (https://aihubmix.com/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Aihubmix,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let aihubmix = Aihubmix::kimi_k2_0905();

    let response = LanguageModelRequest::builder()
        .model(aihubmix)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Aihubmix::builder()`

### API Key

```rust
let aihubmix = Aihubmix::<KimiK20905>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `AIHUBMIX_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let aihubmix = Aihubmix::<KimiK20905>::builder()
    .base_url("https://aihubmix.com/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let aihubmix = Aihubmix::<KimiK20905>::builder()
    .provider_name("Aihubmix")
    .build()?;
```

### Full Custom Configuration Example

```rust
let aihubmix = Aihubmix::<KimiK20905>::builder()
    .api_key("your-api-key")
    .base_url("https://aihubmix.com/v1")
    .provider_name("Aihubmix")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Aihubmix;

// Specify model as a string at runtime
let aihubmix = Aihubmix::model_name("Kimi-K2-0905");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Aihubmix,
};

let aihubmix = Aihubmix::<DynamicModel>::builder()
	.model_name("Kimi-K2-0905")
	.api_key("your-api-key")
	.base_url("https://aihubmix.com/v1")
	.provider_name("Aihubmix")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Aihubmix::kimi_k2_0905()`.
</Callout>
