title: Abacus

AISDK provides first-class support for Abacus with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Abacus provider feature:

```bash
cargo add aisdk --features abacus
```

This installs AISDK with the Abacus provider enabled. Once you have enabled the Abacus provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Abacus::model_name()`, where **model_name** is the Abacus model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Abacus;

let abacus = Abacus::qwen_qwq_32b();
```

This initializes the provider with:

* Model: `"Qwen/QwQ-32B"`
* API key from environment (if set with `ABACUS_API_KEY`)
* Abacus's default base URL (https://routellm.abacus.ai/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Abacus,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let abacus = Abacus::qwen_qwq_32b();

    let response = LanguageModelRequest::builder()
        .model(abacus)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Abacus::builder()`

### API Key

```rust
let abacus = Abacus::<QwenQwq32b>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `ABACUS_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let abacus = Abacus::<QwenQwq32b>::builder()
    .base_url("https://routellm.abacus.ai/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let abacus = Abacus::<QwenQwq32b>::builder()
    .provider_name("Abacus")
    .build()?;
```

### Full Custom Configuration Example

```rust
let abacus = Abacus::<QwenQwq32b>::builder()
    .api_key("your-api-key")
    .base_url("https://routellm.abacus.ai/v1")
    .provider_name("Abacus")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Abacus;

// Specify model as a string at runtime
let abacus = Abacus::model_name("Qwen/QwQ-32B");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Abacus,
};

let abacus = Abacus::<DynamicModel>::builder()
    .model_name("Qwen/QwQ-32B")
    .api_key("your-api-key")
		.base_url("https://routellm.abacus.ai/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Abacus::qwen_qwq_32b()`.
</Callout>
