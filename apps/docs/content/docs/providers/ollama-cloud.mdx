title: OllamaCloud
---

AISDK provides first-class support for OllamaCloud with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the OllamaCloud provider feature:

```bash
cargo add aisdk --features ollama-cloud
```

This installs AISDK with the OllamaCloud provider enabled. Once you have enabled the OllamaCloud provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `OllamaCloud::model_name()`, where **model_name** is the OllamaCloud model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::OllamaCloud;

let ollama-cloud = OllamaCloud::cogito_2_1_671b();
```

This initializes the provider with:

* Model: `"cogito-2.1:671b"`
* API key from environment (if set with `OLLAMA_API_KEY`)
* OllamaCloud's default base URL (https://ollama.com/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::OllamaCloud,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let ollama-cloud = OllamaCloud::cogito_2_1_671b();

    let response = LanguageModelRequest::builder()
        .model(ollama-cloud)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `OllamaCloud::builder()`

### API Key

```rust
let ollama-cloud = OllamaCloud::<Cogito21671b>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `OLLAMA_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let ollama-cloud = OllamaCloud::<Cogito21671b>::builder()
    .base_url("https://ollama.com/v1")
    .build()?;
```

### Path (Full URL Override)

Use `.path(...)` to override the full request URL instead of only the base URL.

```rust
let ollama-cloud = OllamaCloud::<Cogito21671b>::builder()
    .path("https://full-url.example/v1/chat/completions")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let ollama-cloud = OllamaCloud::<Cogito21671b>::builder()
    .provider_name("OllamaCloud")
    .build()?;
```

### Full Custom Configuration Example

```rust
let ollama-cloud = OllamaCloud::<Cogito21671b>::builder()
    .api_key("your-api-key")
    .base_url("https://ollama.com/v1")
    .path("https://full-url.example/v1/chat/completions")
    .provider_name("OllamaCloud")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::OllamaCloud;

// Specify model as a string at runtime
let ollama-cloud = OllamaCloud::model_name("cogito-2.1:671b");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::OllamaCloud,
};

let ollama-cloud = OllamaCloud::<DynamicModel>::builder()
	.model_name("cogito-2.1:671b")
	.api_key("your-api-key")
	.base_url("https://ollama.com/v1")
	.path("https://full-url.example/v1/chat/completions")
	.provider_name("OllamaCloud")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `OllamaCloud::cogito_2_1_671b()`.
</Callout>
