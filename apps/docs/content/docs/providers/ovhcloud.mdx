AISDK provides first-class support for Ovhcloud with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Ovhcloud provider feature:

```bash
cargo add aisdk --features ovhcloud
```

This installs AISDK with the Ovhcloud provider enabled. Once you have enabled the Ovhcloud provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Ovhcloud::model_name()`, where **model_name** is the Ovhcloud model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Ovhcloud;

let ovhcloud = Ovhcloud::deepseek_r1_distill_llama_70b();
```

This initializes the provider with:

* Model: `"deepseek-r1-distill-llama-70b"`
* API key from environment (if set with `OVHCLOUD_API_KEY`)
* Ovhcloud's default base URL (https://oai.endpoints.kepler.ai.cloud.ovh.net/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Ovhcloud,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let ovhcloud = Ovhcloud::deepseek_r1_distill_llama_70b();

    let response = LanguageModelRequest::builder()
        .model(ovhcloud)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Ovhcloud::builder()`

### API Key

```rust
let ovhcloud = Ovhcloud::<DeepseekR1DistillLlama70b>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `OVHCLOUD_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let ovhcloud = Ovhcloud::<DeepseekR1DistillLlama70b>::builder()
    .base_url("https://oai.endpoints.kepler.ai.cloud.ovh.net/v1")
    .build()?;
```

### Path (Full URL Override)

Use `.path(...)` to override the full request URL instead of only the base URL.

```rust
let ovhcloud = Ovhcloud::<DeepseekR1DistillLlama70b>::builder()
    .path("https://full-url.example/v1/chat/completions")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let ovhcloud = Ovhcloud::<DeepseekR1DistillLlama70b>::builder()
    .provider_name("Ovhcloud")
    .build()?;
```

### Full Custom Configuration Example

```rust
let ovhcloud = Ovhcloud::<DeepseekR1DistillLlama70b>::builder()
    .api_key("your-api-key")
    .base_url("https://oai.endpoints.kepler.ai.cloud.ovh.net/v1")
    .path("https://full-url.example/v1/chat/completions")
    .provider_name("Ovhcloud")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Ovhcloud;

// Specify model as a string at runtime
let ovhcloud = Ovhcloud::model_name("deepseek-r1-distill-llama-70b");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Ovhcloud,
};

let ovhcloud = Ovhcloud::<DynamicModel>::builder()
	.model_name("deepseek-r1-distill-llama-70b")
	.api_key("your-api-key")
	.base_url("https://oai.endpoints.kepler.ai.cloud.ovh.net/v1")
	.path("https://full-url.example/v1/chat/completions")
	.provider_name("Ovhcloud")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Ovhcloud::deepseek_r1_distill_llama_70b()`.
</Callout>
