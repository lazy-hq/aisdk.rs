title: Xiaomi
---

AISDK provides first-class support for Xiaomi with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Xiaomi provider feature:

```bash
cargo add aisdk --features xiaomi
```

This installs AISDK with the Xiaomi provider enabled. Once you have enabled the Xiaomi provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Xiaomi::model_name()`, where **model_name** is the Xiaomi model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Xiaomi;

let xiaomi = Xiaomi::mimo_v2_flash();
```

This initializes the provider with:

* Model: `"mimo-v2-flash"`
* API key from environment (if set with `XIAOMI_API_KEY`)
* Xiaomi's default base URL (https://api.xiaomimimo.com/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Xiaomi,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let xiaomi = Xiaomi::mimo_v2_flash();

    let response = LanguageModelRequest::builder()
        .model(xiaomi)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Xiaomi::builder()`

### API Key

```rust
let xiaomi = Xiaomi::<MimoV2Flash>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `XIAOMI_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let xiaomi = Xiaomi::<MimoV2Flash>::builder()
    .base_url("https://api.xiaomimimo.com/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let xiaomi = Xiaomi::<MimoV2Flash>::builder()
    .provider_name("Xiaomi")
    .build()?;
```

### Full Custom Configuration Example

```rust
let xiaomi = Xiaomi::<MimoV2Flash>::builder()
    .api_key("your-api-key")
    .base_url("https://api.xiaomimimo.com/v1")
    .provider_name("Xiaomi")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Xiaomi;

// Specify model as a string at runtime
let xiaomi = Xiaomi::model_name("mimo-v2-flash");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Xiaomi,
};

let xiaomi = Xiaomi::<DynamicModel>::builder()
    .model_name("mimo-v2-flash")
    .api_key("your-api-key")
		.base_url("https://api.xiaomimimo.com/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Xiaomi::mimo_v2_flash()`.
</Callout>
