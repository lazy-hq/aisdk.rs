AISDK provides first-class support for Stackit with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Stackit provider feature:

```bash
cargo add aisdk --features stackit
```

This installs AISDK with the Stackit provider enabled. Once you have enabled the Stackit provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Stackit::model_name()`, where **model_name** is the Stackit model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Stackit;

let stackit = Stackit::qwen_qwen3_vl_235b_a22b_instruct_fp8();
```

This initializes the provider with:

* Model: `"Qwen/Qwen3-VL-235B-A22B-Instruct-FP8"`
* API key from environment (if set with `STACKIT_API_KEY`)
* Stackit's default base URL (https://api.openai-compat.model-serving.eu01.onstackit.cloud/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Stackit,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let stackit = Stackit::qwen_qwen3_vl_235b_a22b_instruct_fp8();

    let response = LanguageModelRequest::builder()
        .model(stackit)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Stackit::builder()`

### API Key

```rust
let stackit = Stackit::<QwenQwen3Vl235bA22bInstructFp8>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `STACKIT_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let stackit = Stackit::<QwenQwen3Vl235bA22bInstructFp8>::builder()
    .base_url("https://api.openai-compat.model-serving.eu01.onstackit.cloud/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let stackit = Stackit::<QwenQwen3Vl235bA22bInstructFp8>::builder()
    .provider_name("Stackit")
    .build()?;
```

### Full Custom Configuration Example

```rust
let stackit = Stackit::<QwenQwen3Vl235bA22bInstructFp8>::builder()
    .api_key("your-api-key")
    .base_url("https://api.openai-compat.model-serving.eu01.onstackit.cloud/v1")
    .provider_name("Stackit")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Stackit;

// Specify model as a string at runtime
let stackit = Stackit::model_name("Qwen/Qwen3-VL-235B-A22B-Instruct-FP8");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Stackit,
};

let stackit = Stackit::<DynamicModel>::builder()
    .model_name("Qwen/Qwen3-VL-235B-A22B-Instruct-FP8")
    .api_key("your-api-key")
		.base_url("https://api.openai-compat.model-serving.eu01.onstackit.cloud/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Stackit::qwen_qwen3_vl_235b_a22b_instruct_fp8()`.
</Callout>
