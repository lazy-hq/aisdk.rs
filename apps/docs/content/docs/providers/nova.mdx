AISDK provides first-class support for Nova with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Nova provider feature:

```bash
cargo add aisdk --features nova
```

This installs AISDK with the Nova provider enabled. Once you have enabled the Nova provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Nova::model_name()`, where **model_name** is the Nova model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Nova;

let nova = Nova::nova_2_lite_v1();
```

This initializes the provider with:

* Model: `"nova-2-lite-v1"`
* API key from environment (if set with `NOVA_API_KEY`)
* Nova's default base URL (https://api.nova.amazon.com/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Nova,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let nova = Nova::nova_2_lite_v1();

    let response = LanguageModelRequest::builder()
        .model(nova)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Nova::builder()`

### API Key

```rust
let nova = Nova::<Nova2LiteV1>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `NOVA_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let nova = Nova::<Nova2LiteV1>::builder()
    .base_url("https://api.nova.amazon.com/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let nova = Nova::<Nova2LiteV1>::builder()
    .provider_name("Nova")
    .build()?;
```

### Full Custom Configuration Example

```rust
let nova = Nova::<Nova2LiteV1>::builder()
    .api_key("your-api-key")
    .base_url("https://api.nova.amazon.com/v1")
    .provider_name("Nova")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Nova;

// Specify model as a string at runtime
let nova = Nova::model_name("nova-2-lite-v1");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Nova,
};

let nova = Nova::<DynamicModel>::builder()
    .model_name("nova-2-lite-v1")
    .api_key("your-api-key")
		.base_url("https://api.nova.amazon.com/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Nova::nova_2_lite_v1()`.
</Callout>
