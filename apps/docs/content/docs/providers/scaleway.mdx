AISDK provides first-class support for Scaleway with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Scaleway provider feature:

```bash
cargo add aisdk --features scaleway
```

This installs AISDK with the Scaleway provider enabled. Once you have enabled the Scaleway provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Scaleway::model_name()`, where **model_name** is the Scaleway model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Scaleway;

let scaleway = Scaleway::bge_multilingual_gemma2();
```

This initializes the provider with:

* Model: `"bge-multilingual-gemma2"`
* API key from environment (if set with `SCALEWAY_API_KEY`)
* Scaleway's default base URL (https://api.scaleway.ai/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Scaleway,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let scaleway = Scaleway::bge_multilingual_gemma2();

    let response = LanguageModelRequest::builder()
        .model(scaleway)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Scaleway::builder()`

### API Key

```rust
let scaleway = Scaleway::<BgeMultilingualGemma2>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `SCALEWAY_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let scaleway = Scaleway::<BgeMultilingualGemma2>::builder()
    .base_url("https://api.scaleway.ai/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let scaleway = Scaleway::<BgeMultilingualGemma2>::builder()
    .provider_name("Scaleway")
    .build()?;
```

### Full Custom Configuration Example

```rust
let scaleway = Scaleway::<BgeMultilingualGemma2>::builder()
    .api_key("your-api-key")
    .base_url("https://api.scaleway.ai/v1")
    .provider_name("Scaleway")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Scaleway;

// Specify model as a string at runtime
let scaleway = Scaleway::model_name("bge-multilingual-gemma2");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Scaleway,
};

let scaleway = Scaleway::<DynamicModel>::builder()
	.model_name("bge-multilingual-gemma2")
	.api_key("your-api-key")
	.base_url("https://api.scaleway.ai/v1")
	.provider_name("Scaleway")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Scaleway::bge_multilingual_gemma2()`.
</Callout>
