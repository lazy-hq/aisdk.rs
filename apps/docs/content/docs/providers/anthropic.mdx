AISDK provides first-class support for Anthropic with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Anthropic provider feature:

```bash
cargo add aisdk --features anthropic
```

This installs AISDK with the Anthropic provider enabled. Once you have enabled the Anthropic provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Anthropic::model_name()`, where **model_name** is the Anthropic model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Anthropic;

let anthropic = Anthropic::claude_3_5_haiku_20241022();
```

This initializes the provider with:

* Model: `"claude-3-5-haiku-20241022"`
* API key from environment (if set with `ANTHROPIC_API_KEY`)
* Anthropic's default base URL (https://api.anthropic.com/v1/)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Anthropic,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let anthropic = Anthropic::claude_3_5_haiku_20241022();

    let response = LanguageModelRequest::builder()
        .model(anthropic)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Anthropic::builder()`

### API Key

```rust
let anthropic = Anthropic::<Claude35Haiku20241022>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `ANTHROPIC_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let anthropic = Anthropic::<Claude35Haiku20241022>::builder()
    .base_url("https://api.anthropic.com/v1/")
    .build()?;
```

### Path (Full URL Override)

Use `.path(...)` to override the full request URL instead of only the base URL.

```rust
let anthropic = Anthropic::<Claude35Haiku20241022>::builder()
    .path("https://full-url.example/v1/chat/completions")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let anthropic = Anthropic::<Claude35Haiku20241022>::builder()
    .provider_name("Anthropic")
    .build()?;
```

### Full Custom Configuration Example

```rust
let anthropic = Anthropic::<Claude35Haiku20241022>::builder()
    .api_key("your-api-key")
    .base_url("https://api.anthropic.com/v1/")
    .path("https://full-url.example/v1/chat/completions")
    .provider_name("Anthropic")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Anthropic;

// Specify model as a string at runtime
let anthropic = Anthropic::model_name("claude-3-5-haiku-20241022");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Anthropic,
};

let anthropic = Anthropic::<DynamicModel>::builder()
	.model_name("claude-3-5-haiku-20241022")
	.api_key("your-api-key")
	.base_url("https://api.anthropic.com/v1/")
	.path("https://full-url.example/v1/chat/completions")
	.provider_name("Anthropic")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Anthropic::claude_3_5_haiku_20241022()`.
</Callout>
