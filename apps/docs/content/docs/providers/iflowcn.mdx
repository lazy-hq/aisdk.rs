AISDK provides first-class support for Iflowcn with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Iflowcn provider feature:

```bash
cargo add aisdk --features iflowcn
```

This installs AISDK with the Iflowcn provider enabled. Once you have enabled the Iflowcn provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Iflowcn::model_name()`, where **model_name** is the Iflowcn model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Iflowcn;

let iflowcn = Iflowcn::deepseek_r1();
```

This initializes the provider with:

* Model: `"deepseek-r1"`
* API key from environment (if set with `IFLOW_API_KEY`)
* Iflowcn's default base URL (https://apis.iflow.cn/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Iflowcn,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let iflowcn = Iflowcn::deepseek_r1();

    let response = LanguageModelRequest::builder()
        .model(iflowcn)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Iflowcn::builder()`

### API Key

```rust
let iflowcn = Iflowcn::<DeepseekR1>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `IFLOW_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let iflowcn = Iflowcn::<DeepseekR1>::builder()
    .base_url("https://apis.iflow.cn/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let iflowcn = Iflowcn::<DeepseekR1>::builder()
    .provider_name("Iflowcn")
    .build()?;
```

### Full Custom Configuration Example

```rust
let iflowcn = Iflowcn::<DeepseekR1>::builder()
    .api_key("your-api-key")
    .base_url("https://apis.iflow.cn/v1")
    .provider_name("Iflowcn")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Iflowcn;

// Specify model as a string at runtime
let iflowcn = Iflowcn::model_name("deepseek-r1");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Iflowcn,
};

let iflowcn = Iflowcn::<DynamicModel>::builder()
	.model_name("deepseek-r1")
	.api_key("your-api-key")
	.base_url("https://apis.iflow.cn/v1")
	.provider_name("Iflowcn")
	.build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Iflowcn::deepseek_r1()`.
</Callout>
