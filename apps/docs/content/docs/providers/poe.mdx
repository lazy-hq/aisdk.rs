AISDK provides first-class support for Poe with fully typed model APIs.
Model capabilities are enforced at compile time using Rust's type system.
This prevents model capability mismatches and guarantees the selected model is valid for the task (e.g. tool calling).


## Installation

Enable the Poe provider feature:

```bash
cargo add aisdk --features poe
```

This installs AISDK with the Poe provider enabled. Once you have enabled the Poe provider, you can use all aisdk <Link href="/docs#core-features">features</Link> with it.

## Create a Provider Instance

To create a provider instance, call `Poe::model_name()`, where **model_name** is the Poe model you want to use.
Model names are exposed as snake-case methods.

```rust
use aisdk::providers::Poe;

let poe = Poe::anthropic_claude_haiku_3();
```

This initializes the provider with:

* Model: `"anthropic/claude-haiku-3"`
* API key from environment (if set with `POE_API_KEY`)
* Poe's default base URL (https://api.poe.com/v1)

## Basic Text Generation

Example using [LanguageModelRequest](https://docs.rs/aisdk/latest/aisdk/core/struct.LanguageModelRequest.html) for text generation.

```rust
use aisdk::{
    core::LanguageModelRequest,
    providers::Poe,
};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {

    let poe = Poe::anthropic_claude_haiku_3();

    let response = LanguageModelRequest::builder()
        .model(poe)
        .prompt("Write a short poem about Rust.")
        .build()
        .generate_text()
        .await?;

    println!("Response text: {:?}", response.text());
    Ok(())
}
```

## Provider Settings

You can customize provider configuration using `Poe::builder()`

### API Key

```rust
let poe = Poe::<AnthropicClaudeHaiku3>::builder()
    .api_key("your-api-key")
    .build()?;
```

If not specified, AISDK uses the `POE_API_KEY` environment variable.

### Base URL

Useful when routing through a proxy, gateway, or self-hosted compatible endpoint.

```rust
let poe = Poe::<AnthropicClaudeHaiku3>::builder()
    .base_url("https://api.poe.com/v1")
    .build()?;
```

### Provider Name

For logging, analytics, and observability.

```rust
let poe = Poe::<AnthropicClaudeHaiku3>::builder()
    .provider_name("Poe")
    .build()?;
```

### Full Custom Configuration Example

```rust
let poe = Poe::<AnthropicClaudeHaiku3>::builder()
    .api_key("your-api-key")
    .base_url("https://api.poe.com/v1")
    .provider_name("Poe")
    .build()?;
```

## Dynamic Model Selection

For runtime model selection (e.g., loading models from config files), use `DynamicModel`:

### Using model_name() Method with Default Settings

```rust
use aisdk::providers::Poe;

// Specify model as a string at runtime
let poe = Poe::model_name("anthropic/claude-haiku-3");
```

### Using Builder Pattern with Custom Settings

```rust
use aisdk::{
    core::DynamicModel,
    providers::Poe,
};

let poe = Poe::<DynamicModel>::builder()
    .model_name("anthropic/claude-haiku-3")
    .api_key("your-api-key")
		.base_url("https://api.poe.com/v1")
    .build()?;
```

<Callout type="warn">
**Warning**: When using `DynamicModel`, model capabilities are **not validated at compile time**. 
This means there's no guarantee the model supports requested features (e.g., tool calls, structured output).
For compile-time safety, use the typed methods like `Poe::anthropic_claude_haiku_3()`.
</Callout>
